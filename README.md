## 🧠 How I Think About Code

```
Good code isn't clever code.
Good code is code your future self
can read at 2am during a production incident
and immediately understand.
```

- **Architecture first** — structure before features, always
- **Layers matter** — presentation never knows about databases; domain never imports Retrofit
- **Interfaces over implementations** — swap providers, not entire modules
- **One responsibility** — if you can't summarise a class in one sentence, split it
